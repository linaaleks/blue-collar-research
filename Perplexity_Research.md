# \# Задание на Deep Research: Рынок высококвалифицированных синих воротничков в РФ

## 1. Контекст и цель

### Роль агента

Аналитик рынка труда и образовательных технологий с фокусом на инженерные специальности и EdTech.

### Цель исследования

Комплексная оценка рынка высококвалифицированных синих воротничков в РФ для принятия решения о запуске образовательного продукта. Исследование должно ответить на вопросы:

1. Какова структура и объём рынка синих воротничков в РФ (B2C/B2B/B2G)?
2. Насколько зрелый сегмент высококвалифицированных синих воротничков?
3. Как соотносится рынок IT и инженерных специальностей?
4. Какой прогноз развития рынка на 3-5 лет?
5. Кто ключевые игроки в обучении синих воротничков (РФ и международка)?
6. Какие технологии используются для обучения (VR, симуляторы, онлайн)?
7. Какие профессии оптимальны для старта образовательного продукта?

### Ограничения

- **География:** Фокус — РФ; международный опыт — для бенчмаркинга и прогнозов
- **Временной период:** 2023-2026 (актуальные данные), прогнозы до 2030
- **Язык источников:** RU, EN
- **Язык отчёта:** RU

## 2. Scope

### Что включаем

**Рынок труда:**

- Вакансии и резюме синих воротничков в РФ (HH, Авито, SuperJob)
- Высококвалифицированные специальности: техники ЦОД, ОВИК, BIM-строители, промышленные электрики, монтажники, операторы ЧПУ, техники робототехники
- Сравнение с IT-рынком (junior/middle позиции)
- Зарплатная аналитика, динамика год к году

**Образовательные решения:**

- Компании, обучающие синих воротничков (B2C/B2B/B2G)
- VR/AR тренажёры для инженерных специальностей
- Онлайн-симуляторы и тренажёры (ROS, Gazebo, специализированное ПО)
- Корпоративные программы обучения

**Технологии:**

- VR/AR платформы для обучения
- Симуляторы робототехники (ROS/ROS2, Gazebo, SLAM)
- Цифровые двойники и BIM
- Тренажёры для электриков, ОВИК, ЦОД

**Инфраструктура будущего:**

- Строительство ЦОД и AI-инфраструктуры
- Облачные системы и их обслуживание
- Physical AI: роботы-курьеры, беспилотный транспорт
- Новая энергетика


### Что НЕ включаем

- Массовые низкоквалифицированные профессии (грузчики, уборщики, курьеры без техники)
- Традиционное ПТУ/колледж образование без инновационной составляющей
- Офисные профессии (белые воротнички)
- Медицинские специальности


### Исключения из исключений

- Курьеры — включать, если речь о операторах/техниках роботов-курьеров
- Колледжи — включать, если есть партнёрство с EdTech или VR-обучение

## 3. Eligibility (критерии отбора)

### Для кейсов компаний/продуктов

- Соответствие scope
- Данные не старше 2022 года
- Минимум 2 источника, из них ≥1 P1
- Для международных — наличие метрик или значимый масштаб


### Для статистики рынка

- Официальные источники (Росстат, Минтруд, HH.ru аналитика)
- Отраслевые отчёты (Data Insight, РБК, Коммерсант)
- Международные отчёты (McKinsey, Gartner, Statista) — для бенчмарка


### Negative cases

Включать если:

- Компания закрылась/приостановила работу с публичным объяснением причин
- Технология не прижилась — есть анализ почему

- Техники ЦОД: количество вакансий, зарплаты, требования
- ОВИК: количество вакансий, зарплаты, требования
- BIM-строители: количество вакансий, зарплаты, требования

- Количество вакансий/откликов

- Strivr, Interplay Learning, Transfr — профиль, сегмент, метрики
- Другие значимые игроки

**Q3.2** Кто основные игроки в РФ?

- VR-компании: Varwin, SIKE, PromVR, ModumLab, КРОК VR, Vizzion

- VR/AR платформы
- Программные симуляторы (ROS, Gazebo, SLAM, path planning)
- Специализированное ПО производителей (FANUC, KUKA, Universal Robots)

**Q4.2** Насколько эти технологии доступны в РФ?

- VR-решения

**Q4.4** Какова стоимость разработки/покупки тренажёров?

- Создание своими силами
- Покупка готовых решений
- Кастомизация существующих


### Блок 5: Выбор профессий для старта

**Q5.1** Какие профессии оптимальны для запуска образовательного продукта?

Критерии оценки:

- Рынок уже есть, ЗП не уступает IT
- Резюме на вакансию меньше чем в IT (дефицит кадров)
- Синергия с Яндексом (ДЦ, роботы, беспилотники)
- Практику можно закрыть существующими тренажёрами или заказать в разумные сроки/бюджеты

**Q5.2** Оценка гипотез по профессиям:

- Специалист по эксплуатации инженерных систем ЦОД
- Промышленный электрик (высоковольтные сети AI-кластеров)
- Монтажник HVAC (прецизионное охлаждение)
- BIM-строитель (цифровые двойники)

## 7. Выходной формат

### Формат документа: Markdown (.md)

### Структура отчёта

1. **Legend** — все теги и шкалы с расшифровками (копия из задания)
2. **Methodology** — как искал, какие запросы, ограничения
3. **Casebook** — кейсы компаний/продуктов C01...Cxx по шаблону
4. **Market Data** — статистика рынка с источниками
5. **Observed Patterns** — паттерны со ссылками на [Cxx] и [Sxx], без выводов
6. **Appendix — Sources** — список источников [Sxx]

### Шаблон карточки (Casebook)

```markdown
### [Cxx] — [Название]: [Краткое описание]


**Базовая информация**
- Название: ...
- Регион: [TAG] — расшифровка
- Год основания/запуска: ...
- Статус: ACT/PAU/DIS — расшифровка


**Классификация**
- Сегмент: [B2C/B2B/B2G] — расшифровка
- Тип решения: [TAG] — расшифровка
- Профессиональный фокус: [TAG] — расшифровка
- Зрелость результатов: [R0-R3] — расшифровка


**Что это / как работает**
[2-3 абзаца]


**Результаты и метрики**
[если есть]


**Финансы**
[выручка, инвестиции — если есть]


**Источники**
[S001], [S002], ...
```

## 8. Observed Patterns

Паттерны группировать по:

- Технологическим трендам
- Региональным особенностям
- Бизнес-моделям
- Барьерам и драйверам роста

Каждый паттерн — со ссылками на кейсы [Cxx] и/или источники [Sxxx].
`[Inference]` — для собственных выводов агента.

---

## 9. Appendix — Sources

Формат:

```
[S001]: P1, [https://full.url/path/to/page](https://full.url/path/to/page), "Название источника"
[S002]: P2, [https://another.url/article](https://another.url/article)
```
